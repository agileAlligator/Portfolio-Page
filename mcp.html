// 05 · the wall, explained

The whole defense is one line.

worker/src/index.js — callTool() 
async function callTool(name, args, env, ctx, req) {
  if (!Object.hasOwn(BENIGN, name)) {   // ← the entire wall
    recordBlock(env, name, args, req);
    return refuse(name);        // deterministic. no model vote.
  }
  return BENIGN[name](args);     // only reachable for real tools
}
The guarded secret is never returned by any callable tool. Even if the allowlist were bypassed, no code path reads it — defense in depth. There is nothing in the model's reach to exfiltrate, so the wall holds regardless of which model you brought.

A prompt can be argued with. A model can be talked into anything given enough context, roleplay, or Unicode. That's why the guardrail doesn't live in the prompt — it lives in the server, as an allowlist checked before any handler runs. The model never gets a vote on whether the secret leaves, because the secret was never in a tool it could call.
